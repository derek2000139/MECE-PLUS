# 03_dynamic_scanner 规范

本模块参考 Shell International 长期采用的 **Scenario Planning (情景规划)** 与 STEEP 宏观趋势分析框架。

## 📥 输入格式

- 读取 `session.original_question`、`session.project_cycle` 和当前所有的 MECE 节点。

## ⚙️ 执行标准

### 1. 时间窗口选择（根据项目周期自动调整）

首先根据 `session.project_cycle` 确定时间窗口：

| 项目周期 | 时间窗口 | 分析侧重点 |
|---------|---------|-----------|
| 超短期 (<1月) | <1月 | 快速执行、资源调配、风险应对 |
| 短期 (1-3月) | 1-3月 | 近期行动计划、里程碑规划 |
| 中期 (3-6月) | 3-6月 | 资源规划、风险预案、中期调整 |
| 长期 (>6月) | 6-12月 | 宏观趋势、战略规划、情景规划 |

**跳过条件**：如果项目周期为超短期 (<1月)，可根据问题性质判断是否需要跳过本工具，直接进入假设驱动分析。

### 2. 趋势分析（根据时间窗口调整）

#### 超短期/短期项目（<3月）：
- 重点关注**运营层面**的变化
- 趋势分析简化为：关键风险点、资源约束、执行里程碑
- 不做复杂情景规划

#### 中期/长期项目（>3月）：
- 进行完整的 **STEEP 趋势扫描**
- 评估外部宏观变量在时间窗口内对各大类权重的冲击：
  - **S** (Social / 社会人口)
  - **T** (Technological / 技术跃迁)
  - **E** (Economic / 经济周期)
  - **E** (Environmental / 生态环境)
  - **P** (Political / 政策监管)

### 3. 情景规划（仅中期/长期项目）

对于中期/长期项目，挑选出**相关度最高、且最不确定**的两个核心驱动力，构建 2×2 的未来图景矩阵：

```
                    【不确定轴 2：变量名】
                         (方向 A) 
                            ▲
                            │   情景 1：[命名]
                            │   (高不确定/高影响)
                            │
 (方向 C) ◄─────────────────┼─────────────────► (方向 D) 【不确定轴 1】
                            │
                            │   情景 2：[命名]
                            │   (基准情景)
                            ▼
                         (方向 B)
```

## 📤 输出规范

### ⏳ 时间维度扫描

#### 1. 时间窗口说明

```
项目周期：[超短期/短期/中期/长期]
分析窗口：[<1月/1-3月/3-6月/6-12月]
分析侧重：[快速执行/近期行动/资源规划/战略规划]
```

#### 2. 趋势/风险分析

**中期/长期项目**：节点权重漂移预测

| 节点 ID | 当前权重 | 未来权重 | 变化趋势 | 核心驱动力 |
| :--- | :--- | :--- | :--- | :--- |
| N1 | 高 | 低 | 📉 下降 | 政策准入收紧 (P) |
| N2 | 低 | 极高 | 📈 激增 | AI 技术平权 (T) |

**短期/超短期项目**：关键风险与里程碑

| 关键风险点 | 影响程度 | 应对措施 | 里程碑节点 |
| :--- | :--- | :--- | :--- |
| [风险1] | 高/中/低 | [措施] | [时间点] |
| [风险2] | 高/中/低 | [措施] | [时间点] |

#### 3. 情景规划（仅中期/长期项目）

- **情景 1 [名称] 发生时的应对策略**：[行动方向]
- **早期预警信号 (Early Warning Signals)**：如出现 [可量化指标 X]，则立即触发情景 1 预案。
