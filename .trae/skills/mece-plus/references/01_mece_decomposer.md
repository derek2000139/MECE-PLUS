# 01_mece_decomposer 规范

本模块严格遵循 Barbara Minto 的《金字塔原理》和 McKinsey Issue Tree 规范。

## 📥 输入格式

- 读取 `SKILL.md` 的标准化议题。

## ⚙️ 执行标准

### 1. 结构化问题界定 (SCQA 模型)

在拆解前，必须明确当前问题的背景与冲突：

- **S (Situation)**: 描述当前稳定的背景。
- **C (Complication)**: 引入打破平衡的冲突。
- **Q (Question)**: 引出待拆解的核心问题。
- **A (Answer)**: 当前可设想的最佳路径假说。

### 2. 维度声明（六选一）

严禁无维度盲目拆解。必须在以下维度中选择一个，并说明理由与潜在偏见：

- **要素法** (人、财、物、信息)
- **流程法** (获客 ➔ 转化 ➔ 留存)
- **主体法** (用户、竞争对手、平台、监管)
- **层级法** (战略、策略、执行)
- **公式法** (收入 = 流量 × 转化率 × 客单价)
- **对比法** (内部原因 vs 外部原因)

## 📤 输出规范

按照以下标准 Markdown 格式输出结构树：

### 🧱 MECE 结构树（所选维度：{维度名}）

```
{标准化议题}
 ├── 1️⃣ [大类 A] (边界描述: {仅用一句话界定该类范围})
 │    ├── 子议题 1.1 (Actionable 动作句式)
 │    └── 子议题 1.2
 ├── 2️⃣ [大类 B] (边界描述)
 │    ├── 子议题 2.1
 │    └── 子议题 2.2
 └── 3️⃣ [大类 C] (边界描述)
      └── 子议题 3.1
```

> ⚠️ **边缘议题声明**：[如存在难以归类的点，必须单独列出，评估其重要性，说明为什么不能归入上述大类，严禁使用“其他”]

### 🔗 割裂点识别（供工具二使用）

- 割裂点一：[大类 A] 与 [大类 B] 被人为切分，但它们在 [某场景] 下存在强互作用。
